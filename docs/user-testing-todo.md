# TODO: минимальный функционал для пользовательского тестирования

Цель: CEO создает и назначает задачи, руководители отделов управляют задачами проектов своих teams, а сотрудники выполняют назначенные им задачи.

## Уже реализовано

- CEO может создавать, редактировать и перемещать задачи на всех досках.
- CEO может создавать и редактировать проекты в любом workspace; director компании — в company workspace своей компании.
- Руководитель отдела может создавать, редактировать и перемещать задачи проекта, если его отдел входит в `project.team`.
- У задачи есть автор, несколько assignees и observers.
- У задачи есть comments и discussion с attachments.
- Seed содержит CEO, руководителей отделов, проекты, teams и задачи.

## Минимальная матрица прав

| Действие | CEO | Director своей компании | Chief отдела из project.team | Assignee | Observer |
| --- | --- | --- | --- | --- | --- |
| Создать/редактировать проект | Да | Только в company workspace | Нет | Нет | Нет |
| Создать задачу | Да | Нет | Только в доступном project | Нет | Нет |
| Редактировать задачу | Да | Нет | Только в доступном project | Нет | Нет |
| Назначать assignees/observers | Да | Нет | Только в доступном project | Нет | Нет |
| Менять статус назначенной задачи | Да | Нет | Да | Да | Нет |
| Читать доступную задачу | Да | Да | Да | Да | Да |
| Писать comments/discussion | Да | Да | Да | Да | Да |

## Проверка перед выдачей пользователям

- [ ] CEO создает проект без прямого взаимодействия с `WorkspaceTeam`, создает задачу и назначает сотрудника.
- [ ] Chief из project team создает и редактирует задачу проекта.
- [ ] Chief другого отдела не может изменить задачу проекта.
- [ ] Assignee видит задачу и переводит ее до `Done`, но не меняет ее содержание или участников.
- [ ] Observer видит задачу и collaboration, но не меняет статус.
- [ ] Пользователь без доступа к workspace/project не видит задачу и не может быть назначен.

Уведомления, delete task, password reset и отдельная страница `My tasks` не блокируют первый закрытый тест.
